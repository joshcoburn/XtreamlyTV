# XtreamlyTV 0.5.0

## Android TV platform parity

Android TV now uses the established webOS product experience as its baseline instead of a separate preview layout.

- Rebuilt the Android TV shell to match the webOS sidebar, top bar, spacing, typography, panel treatment, and D-pad focus language.
- Added matching Teal, Graphite, Purple, Pink, and Blue skins.
- Added Home hero, provider shortcuts, recently watched rails, and recently loaded catalog rails.
- Added category-scoped Live TV, Movies, and Series browsers with compact adaptive grids and search over the loaded category.
- Added details for live channels, movies, and series, including favorites, metadata, resume actions, seasons, and episode browsing.
- Added persistent favorites, custom favorite groups, recent items, and resume positions stored locally on the TV.
- Added provider editing, playback format selection, category-cache controls, diagnostics, privacy controls, and history clearing in Settings.
- Updated the Media3 player overlay with resume support, progress persistence, channel navigation, and favorite control.

## Platform behavior

The two applications now share the same product structure and feature expectations while retaining platform-native implementations. Android TV connects directly to providers and therefore does not require the optional webOS CORS bridge.

## Validation status

Repository checks, version synchronization, webOS syntax validation, bridge syntax validation, core tests, patch whitespace checks, and pure Kotlin URL-builder compilation pass in the generated source tree. The Android Compose application still requires a full Gradle compile and physical-TV D-pad/playback pass before the release is tagged.
