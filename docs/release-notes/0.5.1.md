# XtreamlyTV 0.5.1

## Android TV device polish

This release is a physical-TV refinement pass over the Android 0.5 parity candidate, using the current webOS application as the product baseline.

- Shows a native branded splash immediately while the Android process and Compose UI start.
- Keeps configured-provider startup on the loading experience instead of briefly showing provider setup.
- Authenticates first, opens Home sooner, and discovers provider categories concurrently in the background.
- Matches the webOS Home hero wording, recently watched sections, logo lockup, navigation icons, connected status, and clock.
- Opens content details with Watch or Play focused by default.
- Routes Right from a category directly to the first content card instead of the search field.
- Uses fixed catalog rows and columns so initial category and content views do not show partial items.
- Refines favorites group icons, selected-state padding, title/count spacing, and grid geometry.

The Android APK must still be compiled and tested on supported TV hardware before publishing a release artifact.
