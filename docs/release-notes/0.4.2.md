# XtreamlyTV 0.4.2 — Android TV onboarding and startup

XtreamlyTV 0.4.2 turns the first Android TV preview into a usable living-room onboarding experience while retaining the tested 0.4.1 Android build toolchain.

## Android TV

- Added an immediate branded launch window followed by an in-app startup screen while encrypted credentials are loaded.
- Moved encrypted credential initialization off the UI thread to reduce apparent launch delay.
- Rebuilt the provider screen for common Android TV logical layouts, including 960×540 dp and larger canvases.
- Replaced the overlapping logo lockup with a high-resolution standalone mark and responsive typography.
- Kept the Connect action visible at all times with explicit Connecting and Try again states.
- Added local field validation, D-pad-first focus placement, stable error space, HTTP warnings, connection timeouts, and clearer provider errors.
- Normalized provider addresses, accepted addresses without a scheme, and handled pasted `player_api.php` URLs.
- Loaded the three provider category lists concurrently after authentication.
- Added official-density launcher icons and TV banners from mdpi through xxxhdpi, plus an adaptive launcher icon for API 26 and newer.
- Added provider URL unit tests.

## Platform versions

- Android version code: 402
- Android version name: 0.4.2
- webOS package version: 0.4.2

The webOS application behavior is unchanged except for the synchronized release version.
